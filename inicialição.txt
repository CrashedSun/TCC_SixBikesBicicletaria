Terminal 1 — Backend (API):

npm run start:backend

Vai mostrar: [BACKEND] API rodando em http://localhost:8080



Terminal 2 — Frontend (HTML):

npm run start:frontend

Vai mostrar: [FRONTEND] Rodando em http://localhost:3000

Acesse o site pelo navegador em: http://localhost:3000

Como funciona a comunicação
O frontend (auth.js:4) já define API_BASE_URL = 'http://localhost:8080/api'
Todas as requisições usam fetch() para enviar/receber JSON via HTTP
O backend responde apenas JSON (não serve HTML)
O CORS já está habilitado no backend (app.use(cors())) permitindo que o frontend na porta 3000 acesse a API na porta 8080

Modo desenvolvimento (com auto-reload)

# Terminal 1
npm run dev:backend

# Terminal 2
npm run dev:frontend

Como prosseguir (inicialização e uso)

Subir tudo
Na raiz do projeto Six Bikes:
npm run docker:up
Ou:
docker compose up -d --build
Ver logs
npm run docker:logs
Ou logs de um serviço:
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f db
Acessar aplicação
Frontend: http://localhost:3000/index.html
API: http://localhost:8080
Parar ambiente
npm run docker:down
Ou:
docker compose down
Resetar banco (quando precisar recriar do zero)
Atenção: apaga dados persistidos
docker compose down -v
docker compose up -d --build
Observação importante

O script criaçãodb.sql roda apenas na primeira criação do volume do Postgres. Se você alterar esse SQL depois, precisa resetar volume para reaplicar.

Credenciais seed (dinâmicas)

No seed de usuário, a senha inicial NÃO fica hardcoded em hash fixo.
A senha é gerada dinamicamente a partir do e-mail: parte antes do @.
Exemplos:
- proprietario@sixbikes.com -> senha: proprietario
- gerente@sixbikes.com -> senha: gerente
- atendente@sixbikes.com -> senha: atendente
- mecanico@sixbikes.com -> senha: mecanico

Obs.: para perfis de funcionário, o campo trocar_senha inicia como true para forçar troca no primeiro acesso.

Para voltar o sistema para fresh start no Docker, com apenas os dados padrão do seed, faça assim:

Pare e remova os containers + volume do banco
Comando:
docker compose down -v

Suba tudo novamente reconstruindo
Comando:
docker compose up -d --build

(Opcional) confira se os serviços subiram
Comando:
docker compose ps
